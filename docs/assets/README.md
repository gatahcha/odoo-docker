# Documentation assets

Use this folder for **static files** that support the documentation: screenshots of the Apps UI, exported diagrams, icons, or other images.

When you add a file (for example `apps-pins-kanban.png`), reference it from Markdown under `docs/` with a path relative to the document, for example from `docs/features/apps-pins.md`:

```markdown
![Kanban pin control](../assets/apps-pins-kanban.png)
```

Keep filenames descriptive and avoid committing large binary blobs unless your team policy allows it.
